# Nodepad Mobile

Nodepad runs as a native iOS and Android app via [Capacitor](https://capacitorjs.com/), with device-to-device sync powered by IPFS (Helia).

## Prerequisites

- **Node.js 18+**
- **iOS**: macOS with Xcode 15+ and CocoaPods (`sudo gem install cocoapods`)
- **Android**: Android Studio with SDK 24+ and Gradle

## Quick Start

### Web (development)

```bash
npm install
npm run dev
```

### Web (static build)

```bash
npm run build
# Output in out/ — serve with any static file server
```

### iOS

```bash
npm install
npx cap add ios        # First time only — generates the ios/ folder
npm run mobile:ios     # Builds Next.js, syncs to Capacitor, opens Xcode
```

Then in Xcode: select your device/simulator and press Run.

### Android

```bash
npm install
npx cap add android    # First time only — generates the android/ folder
npm run mobile:android # Builds Next.js, syncs to Capacitor, opens Android Studio
```

Then in Android Studio: select your device/emulator and press Run.

### Subsequent builds

After making code changes:

```bash
npm run mobile:build   # Builds Next.js + syncs to native projects
```

Then re-run from Xcode or Android Studio.

## Mobile Touch Controls

The spatial canvas (force-directed graph) supports full touch interaction:

| Gesture | Action |
|---------|--------|
| **Single finger drag** | Pan the canvas |
| **Pinch** | Zoom in/out |
| **Double tap** (empty area) | Open command palette to add a note |
| **Long press** (on a node) | Select/edit the node |
| **Tap** (on a node) | Toggle node selection |

A floating action button (bottom-right on mobile) opens the command palette since there's no keyboard shortcut available.

## IPFS Sync

Nodepad uses [Helia](https://helia.io/) (IPFS in the browser) for peer-to-peer note syncing between devices. No server required.

### How it works

1. Open the **IPFS Sync** panel (via command palette or the "IPFS Sync" command)
2. The Helia node starts automatically in the background (status indicator shows readiness)
3. **Push to IPFS**: Publishes your current notes and gives you a CID (Content Identifier)
4. **Share the CID**: Copy the CID and send it to your other device (paste, QR, message, etc.)
5. **Pull from CID**: On the other device, paste the CID and pull — notes are merged with local notes

### Important notes

- IPFS sync is **additive** — pulling from a CID merges new notes with your existing ones (no overwrites)
- Notes are identified by their unique ID, so pulling the same CID twice won't create duplicates
- The Helia node uses **WebRTC** for direct browser-to-browser connections and **WebSocket** relays as fallback
- CIDs are stored in localStorage so you can reference your last published CID
- IPFS content is **not encrypted** — anyone with the CID can read the notes. For sensitive data, share CIDs privately.

## Architecture

- **Next.js static export** (`output: 'export'`) generates a static site in `out/`
- **Capacitor** wraps the static site in a native WebView for iOS/Android
- **All data** is stored in the device's localStorage — no server, no cloud
- The API route (`/api/fetch-url`) for URL metadata extraction is **not available** in static/mobile builds. URL enrichment gracefully degrades (skipped).
- The `ios/` and `android/` directories are generated by `npx cap add` and are git-ignored
